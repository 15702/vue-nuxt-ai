# 林安个人主页 — 需求文档

> 版本：v1.0
> 日期：2025-06-23
> 状态：第一版原型已完成，待确认

---

## 一、项目概述

### 1.1 项目目标

搭建一个林安的个人主页，核心功能是让访客通过页面了解林安，并能与一个"数字分身"进行实时对话，降低社交沟通成本。

### 1.2 目标用户

| 用户类型 | 访问目的 |
|---|---|
| 朋友 | 了解林安近况 |
| 潜在合作伙伴 | 评估林安的能力方向和作品 |
| 面试官 | 快速了解林安的背景与特长 |

### 1.3 核心价值主张

> 一个能自我介绍、能回答问题的个人主页——访客不需要等人回复，就能获得基本了解。

## 三、页面结构与功能需求

### 3.1 页面结构

```
个人主页
顶部导航栏（粘性）
Hero 区（头像 + 名字 + 一句话介绍 + 状态标识）
关于我（信息卡片区）
  现在在做什么
  擅长的方向
  兴趣领域
  联系方式
数字分身聊天区
  聊天头部（名称 + 状态）
  消息展示区
  快捷提问按钮
  输入框 + 发送按钮
底部（版权 + 社交链接）
```

### 3.2 各模块详细需求

#### 模块 1：顶部导航栏

| 需求项 | 说明 |
|---|---|
| 固定方式 | 吸顶粘性（sticky） |
| 内容 | 首字母标识 + "林安 · 个人主页" |
| 视觉效果 | 毛玻璃半透明背景 |
| 交互 | 无特殊交互 |

#### 模块 2：Hero 区

| 需求项 | 说明 |
|---|---|
| 头像 | 圆形，当前用首字母占位，后续替换为真实照片 |
| 名字 | "林安"，加粗大字 |
| 一句话介绍 | "一个正在学习用 AI 做产品的内容策划" |
| 状态标识 | 蓝色胶囊标签，显示"数字分身在线"，带呼吸动画 |
| 响应式 | 手机端头像和文字垂直居中排列 |

#### 模块 3：关于我 — 信息卡片区

| 卡片 | 内容 |
|---|---|
| 现在在做什么 | 搭个人主页，整理作品集，梳理写作方向 |
| 擅长的方向 | 内容表达 · AI 应用 · 知识整理；喜欢把复杂问题讲成人话 |
| 兴趣领域 | 标签形式：AI 应用、写作、旅行、内容策划 |
| 联系方式 | 引导文字 + 底部社交链接 |

- 布局：2×2 网格，手机端单列
- 交互：hover 微上浮 + 阴影增强

#### 模块 4：数字分身聊天区（核心功能）

**4a. 基础 UI**

| 需求项 | 说明 |
|---|---|
| 聊天头部 | 深蓝背景，显示"林安的数字分身" + 副标题 |
| 消息气泡 | 左右分列，机器人消息靠左灰色，用户消息靠右蓝色 |
| 输入框 | 圆角输入框 + 蓝色发送按钮，支持 Enter 键发送 |
| 消息动画 | 新消息出现时有从下往上淡入的过渡效果 |
| 打字指示器 | 数字分身回复前显示三个跳动圆点 |

**4b. 快捷提问**

预设三个高频问题按钮，点击后自动发送并获得回复：

1. 你现在在做什么？
2. 你有哪些作品？
3. 怎么联系你？

**4c. 数字分身知识库**

数字分身需要掌握以下信息：

| 类别 | 内容 |
|---|---|
| 职业身份 | 内容策划 |
| 近期动态 | 搭个人主页，整理作品集 |
| 擅长方向 | 内容表达、AI 应用、知识整理 |
| 兴趣 | AI 应用、写作、旅行 |
| 特点 | 把复杂问题讲成人话 |

**4d. 预设问答覆盖**

数字分身应能回答以下类型的问题（第一版用关键词匹配实现）：

| 问题类型 | 关键词示例 |
|---|---|
| 现在在做什么 | 做什么、在做、最近、忙、工作 |
| 有哪些作品 | 作品、项目、案例、做过 |
| 怎么联系 | 联系、邮箱、微信、方式 |
| 擅长什么 | 擅长、优势、强项、技能 |
| 兴趣爱好 | 兴趣、爱好、喜欢 |
| 背景经历 | 背景、经历、学历、履历 |
| AI 相关 | AI、人工智能、大模型 |
| 打招呼 | 你好、hi、hello、在吗 |
| 个人介绍 | 谁、是谁、介绍、什么人 |

**4e. 未知问题兜底**

当无法匹配到已知答案时，回复引导性文字，建议联系林安本人。

## 四、设计规范

### 4.1 视觉风格

| 维度 | 规范 |
|---|---|
| 整体风格 | 简约、清爽、略带科技感 |
| 主色调 | 深蓝（#0a1628）+ 白色 |
| 强调色 | 蓝色（#2563eb） |
| 背景色 | 浅灰蓝（#f6f8fc） |
| 圆角 | 统一 16px |
| 不要 | 过度花哨、信息层级不清晰 |

### 4.2 字体

| 用途 | 字体 |
|---|---|
| 中文正文 | Noto Sans SC |
| 标识/首字母 | JetBrains Mono |

### 4.3 响应式要求

| 断点 | 适配 |
|---|---|
| > 640px | 桌面端，卡片 2×2 网格 |
| ≤ 640px | 移动端，卡片单列，Hero 垂直居中 |

## 五、技术方案（第一版）

| 项目 | 方案 |
|---|---|
| 部署形式 | 单个 HTML 文件，可直接浏览器打开 |
| 聊天逻辑 | 前端纯 JavaScript 关键词匹配（无后端） |
| 字体加载 | Google Fonts CDN |
| 依赖 | 零外部依赖，无框架 |
| 兼容性 | 现代浏览器（Chrome / Safari / Edge / Firefox） |

## 七、验收标准

- [ ] 页面在手机和电脑上均能正常显示，布局不破裂
- [ ] 点击快捷提问按钮能触发对话并获得合理回复

- [ ] 整体视觉风格统一，无明显违和元素
