# 技术实现方案（v1.0 原型）

> 来源：`need1.0.md`（刘豪个人主页 + “数字分身”离线关键词匹配聊天）

## 1. 需求拆解（按页面模块）

### 1.1 顶部导航栏（粘性 + 毛玻璃）
- 交互：无特殊交互（仅展示）
- 视觉：吸顶（sticky），毛玻璃半透明背景
- 内容：首字母标识 + “刘豪 · 个人主页”

### 1.2 Hero 区（头像 + 姓名 + 一句话 + 在线状态）
- 内容：
  - 头像：圆形，v1.0 使用首字母占位
  - 名字：粗体“大字”展示“刘豪”
  - 一句话介绍：“一个正在学习用 AI 做产品的内容策划”
  - 状态标识：蓝色胶囊标签，显示“数字分身在线”，带呼吸动画
- 响应式：
  - 手机：头像与文字垂直居中排列

### 1.3 关于我（信息卡片区）
- 布局：2x2 网格（桌面），手机端单列
- 卡片内容：
  - “现在在做什么”：搭个人主页，整理作品集，梳理写作方向
  - “擅长的方向”：内容表达 · AI 应用 · 知识整理；喜欢把复杂问题讲成人话
  - “兴趣领域”：AI 应用、写作、旅行、内容策划（标签形式）
  - “联系方式”：引导文字 + 底部社交链接
- 交互：hover 微上浮 + 阴影增强

### 1.4 数字分身聊天区（核心功能）

#### 4a. 基础 UI
- 聊天头部：深蓝背景，显示“刘豪的数字分身” + 副标题
- 消息气泡：
  - 机器人消息靠左灰色
  - 用户消息靠右蓝色
- 输入框：
  - 圆角输入框 + 蓝色发送按钮
  - 支持 `Enter` 键发送
- 消息动画：新消息出现有从下往上淡入过渡
- 打字指示器：数字分身回复前显示三个跳动圆点

#### 4b. 快捷提问
- 3 个高频问题按钮点击即发送并获得回复：
  1. 你现在在做什么？
  2. 你有哪些作品？
  3. 怎么联系你？

#### 4c. 数字分身知识库（第一版数据）
- 职业身份：内容策划
- 近期动态：搭个人主页，整理作品集
- 擅长方向：内容表达、AI 应用、知识整理
- 兴趣：AI 应用、写作、旅行
- 特点：把复杂问题讲成人话

#### 4d. 预设问答覆盖（关键词匹配）
- 9 类问题类型及关键词示例：
  - 现在在做什么：做什么、在做、最近、忙、工作
  - 有哪些作品：作品、项目、案例、做过
  - 怎么联系：联系、邮箱、微信、方式
  - 擅长什么：擅长、优势、强项、技能
  - 兴趣爱好：兴趣、爱好、喜欢
  - 背景经历：背景、经历、学历、履历
  - AI 相关：AI、人工智能、大模型
  - 打招呼：你好、hi、hello、在吗
  - 个人介绍：谁、是谁、介绍、什么人
- 匹配规则（建议）：
  - 对用户输入做轻量预处理（trim，大小写归一，保留中文）
  - 如果命中多个类别，选择“命中关键词数量最多”的类别（或最高分类别）
  - 未命中则兜底：给引导性文字并建议联系本人

#### 4e. 未知问题兜底
- 未匹配到已知答案：回复引导性文字，建议联系刘豪本人

### 1.5 底部（版权 + 社交链接）
- 版权信息
- 社交链接（供“联系方式”卡片与底部复用）

## 2. 技术选型与约束（基于当前 Nuxt 项目）

- 前端框架：Nuxt 4（Nitro），Vue 3
- UI 库：Element Plus（`element-plus` 已依赖）
- 状态/数据：
  - v1.0 可用页面内 `ref` 管理聊天状态（无需额外状态库）
  - 若后续扩展（多页面/持久化），再引入 Pinia（当前未依赖）
- 实现方式：
  - v1.0 聊天逻辑：纯前端离线关键词匹配（无后端接口）
  - v1.2 后：接入真实 AI API 时再增加 Nitro server 接口（建议）

## 3. 页面与组件拆分建议（可落地的文件结构）

- `app/pages/index.vue`
  - 组装：`<Navbar/> <Hero/> <AboutSection/> <ChatSection/> <Footer/>`
  - 提供数据常量（个人信息、社交链接、知识库、问答映射）
- `app/components/`

  - `Hero.vue`
  - `AboutSection.vue`

    - `TagList.vue`（兴趣标签）
  - `ChatSection.vue`
    - `ChatHeader.vue`
    - `ChatMessages.vue`
    - `ChatBubble.vue`
    - `QuickQuestions.vue`

## 4. 数据模型与关键状态（Chat）

### 4.1 数据结构建议
- `ChatMessage`
  - `id: string`
  - `role: 'user' | 'assistant'`
  - `content: string`
  - `createdAt: number`
  - （可选）`category?: string` 用于调试或埋点

### 4.2 会话状态建议
- `messages: ChatMessage[]`
- `draft: string`（输入框内容）
- `isAssistantTyping: boolean`（打字指示器开关）
- `isSending: boolean`（节流/避免重复发送）

## 5. 聊天逻辑实现（v1.0：关键词匹配）

### 5.1 匹配引擎：`matchQuestion(input) -> answer`
- 输入：用户输入 `input: string`
- 输出：
  - 命中：返回对应类别的回答文本
  - 未命中：返回兜底文本

### 5.2 知识库与问答文本来源
- v1.0 采用“静态字典”：
  - `category -> { keywords: string[], answer: string }`
- 建议把“知识库内容”（职业身份/近期动态/擅长/兴趣/特点）映射到对应类别回答中，避免重复维护。

### 5.3 发送流程（包含打字指示器与动画）
1. 用户点击发送或按 `Enter`
2. 校验：空输入不发送；发送中禁用按钮（防止连点）
3. 将用户消息 push 到 `messages`
4. 设置 `isAssistantTyping = true`
5. 通过 `setTimeout` 模拟机器人思考（例如 600ms-1400ms，后续可调）
6. 调用 `matchQuestion(draft)` 得到答案
7. 将机器人消息 push 到 `messages`
8. 设置 `isAssistantTyping = false`，清空 `draft`
9. 自动滚动到消息底部（`scrollIntoView`/ref 定位末尾元素）

## 6. 交互与样式落地要点

### 6.1 输入框与发送
- `Enter` 发送：
  - 在输入框组件监听 `keydown.enter`
  - 阻止默认换行行为
- 发送按钮：
  - 禁用态：`isSending || isAssistantTyping`

### 6.2 消息气泡布局
- 机器人：左侧灰色气泡
- 用户：右侧蓝色气泡
- 采用 CSS Flex 或 Grid 控制气泡对齐

### 6.3 新消息入场动画
- 在 `ChatBubble` 组件上应用进入动画：
  - `opacity` 从 0 -> 1
  - `transform: translateY(8px) -> translateY(0)`

### 6.4 打字指示器动画
- 三个跳动圆点（CSS keyframes）
- 显示时占位在“机器人消息将出现的位置”

### 6.5 顶部毛玻璃效果
- 使用 CSS 实现：背景半透明 + `backdrop-filter: blur(...)`
- 如果浏览器不支持 `backdrop-filter`：使用更保守的半透明背景色（可作为 fallback）

### 6.6 响应式断点
- 需求给定：
  - `> 640px`：桌面端卡片 2x2 网格
  - `<= 640px`：移动端卡片单列、Hero 垂直居中
- 建议在 `AboutSection` 与 `Hero` 的容器中加入媒体查询。

## 7. 字体与资源加载（Google Fonts）

需求要求：
- 中文正文：`Noto Sans SC`
- 标识/首字母：`JetBrains Mono`

实现建议（Nuxt 方式）：
- 在 `app.vue` 或页面里使用 `useHead` 注入两个字体的 `<link rel="stylesheet" ...>`。
